# [GIT] Cherry Pick
## 언제 사용하나
- 하위 브랜치에서 상위브랜치에 원하는(내가 작업한) 커밋 내용만 반영하고 싶을때.

## 원래 하던 방법(merge)
- 하위 브랜치 : dev, 상위브랜치 : stg
- dev에서 새로운 브랜치 생성 : feat/dev
- feat/dev 에서 수정, 커밋, 푸시
- feat/dev를 dev에 머지
- feat/dev를 stg에 머지

## 체리픽으로 하면...
- 하위 브랜치 : dev, 상위브랜치 : stg
- dev에서 수정, 커밋, 푸시
- stg에서 체리픽

## 장점
- 빨리 할 수 있음, 단순함
- 단점 같은 내용의 커밋을 복수로 생성하기 때문에 이력 관리에 좋진 않음
